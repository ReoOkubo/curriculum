IT
IT : インターネットなどの通信とコンピューターを駆使する情報技術。
ICT : 情報や通信に関連する科学技術の総称。
IoT : 世の中のモノに通信機能を持たせ、自動認識や自動制御、遠隔計測などを行うこと。

OS
OS : システム全体を管理するソフトウェアの一種。
Windows : マイクロ社が開発したOS製品。
macOS : アップル社がMacシリーズ向けに提供している、OS製品。
UNIX : AT&T社ベル研究所で開発されたOS製品。
Linux : UNIX系の一種で誰でも自由に入手や使用、改変、再配布ができる。

言語・環境
マークアップ/フロントエンド : 外部との直接やりとりを行う要素。
HTML : Webページを記述するためのマークアップ言語。
CSS : Webページの配置や見栄えなどを記述する言語。
JavaScript : Webページのプログラムをブラウザ上で実行するためのプログラミング言語の一つ。
サーバーサイド : サーバー側で動く何かを指す。
.NET (C#, VB) : 総合的な開発環境。
Java : OS関係あく、汎用的に使われるプログラミング言語。
Java SE : Javaにおける機能セットの基準の一つで、汎用的な機能をまとめたもの。
Java EE (Jakarta EE) : Javaにおける機能セットの一つで、SEに拡張機能を足して、大規模システム向けの高度な機能をまとめたもの。
Perl :  柔軟性、拡張性の高さが特徴的な高水準のプログラミング言語の一つ
PHP : Webアプリに特化したプログラミング言語。
Ruby : オブジェクト指向プログラミング

サーバーソフト
Apache : 世界的に最も普及しているWebサーバ（HTTPサーバ）ソフトウェアの一つ
nginx : オープンソースのWebサーバソフトウェアの一つ
IIS : Webサーバソフトウェア

SQL
SQL : リレーショナルデータベースの管理や操作を行うための問い合わせ言語の一つ
DB2 : 米IBM社のリレーショナルデータベース管理システム
MySQL : 人気の高いオープンソースのリレーショナルデータベース管理システムの一つ。
Maria DB : MySQL派生として開発されている、オープンソースの関係データベース管理システム
Oracle DB : 米国オラクル が開発・販売している、関係データベース管理システム
PostgreSQL : リレーショナルデータベースの作成や操作、管理ができるオープンソースのデータベース管理システム（DBMS）の一つ
SQLite : オープンソースのリレーショナルデータベース管理システムの一つ
SQL Server : リレーショナルデータベース管理システム。同社のオペレーティングシステム（OS）製品であるWindowsシリーズおよびLinuxで動作する
CRUD : 永続的なデータを取り扱うソフトウェアに要求される4つの基本機能である、データの作成（Create）、読み出し（Read）、更新（Update）、削除（Delete）の頭文字を繋げた語。
トランザクション : 複数の複数の処理をまとめるソフトウェア処理方式の一つ
コミット : 処理や変更などを確定させる、反映させる、といった意味
ロールバック : システム障害やデータの喪失、破損などが起きた際に、以前に正常に稼働していたある時点の状態に戻して復旧を試みること
排他制御 (排他ロック) : 複数の主体が同じ資源を同時に利用すると競合状態が生じる場合に、ある主体が資源を利用している間、別の主体による資源の利用を制限もしくは禁止する仕組み
楽観ロック : データそのものに対してロックは行わずに、更新対象のデータがデータ取得時と同じ状態であることを確認してから更新することで、データの整合性を保証する方式
悲観ロック : 更新対象のデータを取得する際にロックをかけることで、他のトランザクションから更新されないようにする方式
実行計画 : どうしたらより短い時間でSQLを実行できるか、計算して導き出された、具体的な方法を「計画」としてまとめたもの
チューニング : 情報システムやコンピュータ、ソフトウェアなどの設定や構成を調整し、目標の状態に近づけたり、性能を最大限引き出したりする作業を指す
パフォーマンス : コンピュータなどの機器やソフトウェア、システムなどの処理性能や実行速度、通信回線・ネットワークなどの伝送速度・容量などのこと
スロークエリ : データベースにおいて、時間のかかっているSQL(遅いSQL)のこと
クエリログ : クライアントから実行された全ての SQL クエリを出力してくれるログ
IDE（統合開発環境) : ソフトウェア開発に必要なソフトウェアを一つに組み合わせ、同じ操作画面から統一的な操作法で利用できるようにしたソフトウェアパッケージ
Eclipse : Javaを始めとする様々なプログラミング言語によるソフトウェア開発に対応した、オープンソースの統合開発環境（IDE）の一つ
NetBeans : オラクルを中心としたコミュニティにより開発されている、オープンソースの統合開発環境
SpringToolSuite : Spring Bootでの開発に使用する
DBツール : データベース管理タスクに使用できるツール
A5:SQL Mk-2 : データベース開発において生産性を向上させるべく開発されたSQL開発ツール
pgAdmin : Postgres データベースの設計、保守、管理を行なうための汎用ツール
DBeaver : 計42種類のドライバに対応したSQLクライアントツール
MySQL Workbench : MySQLの公式サイトにてMySQL Serverと共に配布されている、データベースの設計・開発・管理を一本で実現している無料ツール
Sequel Pro : データベースをExcelのような表で確認・編集をすることができるMac専用のアプリ
TeamSQL : DBをGUIで管理する為のソフトウェアのひとつ

UI（ユーザーインターフェース）
UI : ユーザーの目に触れる部分使用する部分

ファイル／ディレクトリ
ファイル : コンピュータにおけるデータの管理単位の一つ
拡張子 : パソコンやスマホで扱うファイルの種類を識別するための文字列のこと
ディレクトリ : ファイルなどが入っているもの
ルート・ディレクトリ : ディレクトリの根っこの部分
カレント・ディレクトリ : 今いるディレクトリの部分
パス : ファイルやフォルダの置いてある場所
絶対パス : URL でページを指定する
相対パス : 今いる場所を基準にして目的ファイルの場所を伝える

ネットワーク
LAN : 限られた範囲内にあるコンピュータや通信機器、情報機器などをケーブルや無線電波などで接続し、相互にデータ通信できるようにしたネットワーク
WAN : 遠く離れたエリアとつながったネットワーク
クライアント : サーバからサービスを受ける側
サーバー : サービスを提供する側

各種プロトコル
HTTP : ホームページを移動するときに使う約束事
FTP : インターネットの線を使ってファイルを送受信する仕組み
HTTPS : ホームページを見るときの通信手段で暗号化されている
SSL : インターネット上の通信を暗号化する技術
SSH : ネットワークに接続された機器を遠隔操作し、管理するための手段
ポート、ポート番号 : ネットワークでデータを通信するためのもの
                  通信に使用するプログラムを識別するための番号
HTTP、FTP、SSHの各ポート番号 : HTTP.TCP 80 , FTP.TCP 20 , SSH.TCP 22 
ファイアウォール : インターネットを通して侵入してくる不正なアクセスから守るための防火壁
TCP : 送ったデータが相手に届いたか、その都度確認しながら通信


IPアドレス
IPアドレス : インターネット上に接続された機器が持つナンバー
グローバルIPアドレス : インターネットを利用するときに割り振られるIPアドレス
プライベートIPアドレス : 社内LANなど小規模なネットワークで使用されるIPアドレス

ドメイン
ドメイン : インターネット上に存在するコンピューターやネットワークを識別するための名前
DNS : インターネット上におけるホスト名（FQDN）やドメイン名に対応するIPアドレス情報を管理・運用するシステム
VPN : 仮想サーバを提供する仮想専用サーバサービス

ブリッジ
プロキシサーバー : インターネットに直接接続できないコンピューターに代わり、インターネットに接続し、Webサイトへのアクセスなどを行うサーバー

電子メール
SMTP : メールを相手のメールサーバーまで届ける仕組み
POP : メールサーバーに保存されたメールを、受信者のパソコンなどへダウンロードする役割
IMAP : メールサーバーに保存したままの状態でメールを読むことができる

セッションとクッキー
セッション : 通信の開始から終了までを指す
クッキー : Webサイトとユーザー間でやり取り・保存されるアクセス情報
キャッシュ : よく使うデータを取り出しやすいところに準備しておく仕組み

DB（データベース）について
インデックス : データベースの性能を向上させる方法

テストについて
フロー : コンピュータプログラムを構成する命令や手続き、処理の流れ

チケット管理ツール
タスク : ソフトウェアの実行単位

バージョン管理
Git : 複数の開発者が共同で一つのソフトウェアを開発する際に使われる管理システム
SVN : サーバーにインストールした”リモートリポジトリ”だけが、ソースコードのバージョンを管理
デグレ : 修正したプログラムなどの品質が下がる
マージツール : 複数のプログラムを統合するための機能。
